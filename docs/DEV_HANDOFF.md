# Dev Handoff _개발 전달

## 1. 프로젝트 개요
이 프로젝트는 애견 가구/소품 쇼핑몰 브랜드 `PAVONIQ`를 위한 정적 HTML/CSS/JS 퍼블리싱 작업입니다. Cafe24 쇼핑몰 스킨 적용 가능성을 고려해 페이지별 HTML과 공통 CSS, 공통 JS, 이미지 에셋 폴더로 단순하게 구성되어 있습니다.

현재 단계는 디자인 시안과 퍼블리싱 구조를 맞추는 단계입니다. 실제 상품 데이터, 회원 인증, 결제, 장바구니 저장 같은 서버 기능은 연결되어 있지 않습니다.

## 2. 페이지 구조

## 4. 공통 컴포넌트 구조

### Header
각 HTML 파일 상단에 `topbar`와 `site-header`가 반복됩니다.

- `topbar`: 신규 회원 쿠폰, 무료배송, 카카오톡 포인트 안내
- `.site-header`: 로고, 네비게이션, 검색/로그인/위시리스트/장바구니/모바일 메뉴 아이콘
- `.logo`: `PAVONIQ` 텍스트와 `For the life we love` 서브 카피
- `.main-nav`: 베스트, 강아지, 고양이, 가구, 급식/급수, 수납/정리, 브랜드, 이벤트

Cafe24 적용 시에는 헤더를 공통 레이아웃 또는 include 영역으로 분리하는 것을 권장합니다.

### Footer
각 HTML 파일 하단에 `.site-footer`가 반복됩니다.

- 푸터 브랜드 로고
- 쇼핑 메뉴
- 고객센터 링크
- 멤버십 & 혜택 링크
- 고객센터 정보
- 회사 정보
- copyright

현재 고객센터 번호, 사업자 정보, 통신판매업신고 번호는 샘플 값입니다. 운영 전 실제 정보로 교체해야 합니다.

### Navigation
네비게이션 링크는 대부분 `products.html`, `index.html#brand`, `index.html#membership`로 연결됩니다. 현재 카테고리별 별도 페이지는 분리되어 있지 않으며, 상품 목록 페이지 하나를 중심으로 연결됩니다.

### Product Card
상품 카드는 `.product-card`, `.product-thumb`, `.product-meta`, `.brand`, `.product-name`, `.price` 조합으로 구성됩니다.

- 메인, 상품 목록, 위시리스트, 추천 상품 영역에서 재사용되는 구조입니다.
- 이미지 비율은 CSS의 `.product-thumb`에서 정사각형에 가깝게 유지합니다.
- 하트 버튼은 `.heart` 클래스를 사용하며, JS에서 클릭 시 `on` 상태를 토글합니다.

### CTA Button
CTA는 `.btn`과 `.btn.primary`를 사용합니다.

- `.btn`: 기본 테두리 버튼
- `.btn.primary`: 주요 액션용 포인트 컬러 버튼
- 주요 사용 위치: 컬렉션 보기, 장바구니 담기, 바로 구매하기, 주문하기, 회원가입, 멤버십 안내

## 5. 디자인 방향
- 애견 가구/소품 브랜드에 맞춘 따뜻하고 부드러운 무드입니다.
- 메인 컬러는 브라운 계열과 코랄 포인트 컬러를 사용합니다.
- 상품 이미지가 중심이 되는 쇼핑몰 구조입니다.
- 과한 인터랙션보다 Cafe24 스킨 적용과 유지보수를 고려한 단순한 구조를 우선합니다.
- 카드, 버튼, 탭, 필터, 폼 등 쇼핑몰 기본 UI를 일관된 스타일로 구성합니다.

## 6. 개발 전달 사항

### HTML 파일별 역할
- `index.html`: 브랜드 첫 화면과 상품 탐색 시작점입니다.
- `products.html`: 상품 목록과 필터 UI를 보여주는 카탈로그 화면입니다.
- `product-detail.html`: 단일 상품 상세 화면입니다. 상세페이지 이미지는 `assets/images/detail-page/` 하위 파일을 순서대로 노출합니다.
- `cart.html`: 장바구니 샘플 화면입니다. 수량 조절과 삭제 버튼은 프론트엔드 데모 수준으로 동작합니다.
- `wishlist.html`: 관심상품 샘플 화면입니다. 선택/삭제/장바구니 담기 기능은 실제 데이터와 연결되어 있지 않습니다.
- `login.html`: 로그인 UI 화면입니다. 실제 인증 요청은 없습니다.
- `signup.html`: 회원가입 UI 화면입니다. 실제 회원 생성 요청은 없습니다.

### CSS 관리 방식
- 전체 스타일은 `css/style.css` 한 파일에서 관리합니다.
- 전역 변수는 `:root`에 정의되어 있습니다.
- 주요 변수: `--main`, `--sub`, `--point`, `--bg`, `--line`, `--muted`, `--section-gap`, `--container`, `--radius`, `--shadow`
- 반응형은 하단 media query에서 관리합니다.
- 페이지별 스타일과 공통 컴포넌트 스타일이 한 파일에 함께 있으므로, 운영 단계에서 규모가 커지면 공통/페이지별 CSS 분리를 검토할 수 있습니다.

### JS 사용 영역
`js/main.js`는 현재 다음 UI 동작을 담당합니다.

- 스크롤 시 헤더 상태 변경
- 모바일 메뉴 열기/닫기
- reveal 애니메이션
- 위시리스트 하트 버튼 토글
- 수량 조절 버튼
- 상세 탭 전환
- 메인 프로모션 배너 자동 전환과 도트 제어
- 장바구니 행 삭제와 카운트 변경
- 맨 위로 이동 버튼

실제 상품 데이터, 장바구니 데이터, 로그인 상태와는 연결되어 있지 않습니다.

### 이미지 경로 관리 방식
- 일반 상품 이미지: `assets/images/product-hq-*.jpg`
- 메인 배너 이미지: `assets/images/main-banner-*.png`
- 상세페이지 이미지: `assets/images/detail-page/detail-page-*.png`
- 소셜 로그인 아이콘: `assets/images/social-*.svg`
- 페이지 보조 이미지: `hero-hq.png`, `wishlist-banner.jpg`, `login-lifestyle.jpg`, `signup-lifestyle.jpg` 등

상세페이지 이미지는 `product-detail.html`에서 `detail-page-01.png`부터 `detail-page-13.png`까지 순차적으로 노출됩니다. 13번 이미지는 `1129 x 2258` 크기의 최신 이미지로 교체되어 있습니다.

### Cafe24 스킨 적용 시 분리해야 할 영역
- 공통 상단 프로모션 바
- 공통 헤더
- 공통 푸터
- 상품 카드 반복 영역
- 상품 목록 필터와 정렬 영역
- 상품 상세 정보 영역
- 장바구니 상품 행 반복 영역
- 위시리스트 상품 반복 영역
- 로그인/회원가입 폼 액션 영역

Cafe24 템플릿 변수 또는 반복 구문을 적용할 때는 현재 정적 샘플 데이터를 실제 상품/회원/장바구니 데이터로 치환해야 합니다.

## 7. 반응형 전달 사항
- 기본 구조는 PC 우선입니다.
- `--container` 최대 폭을 기준으로 넓은 화면에서 안정적인 중앙 정렬을 유지합니다.
- 1180px 이하에서 모바일 네비게이션 구조가 적용됩니다.
- 820px 이하에서 상세, 카탈로그, 로그인, 회원가입, 장바구니, 위시리스트 주요 레이아웃은 1열 흐름으로 정리됩니다.
- 520px 이하에서는 탭, 버튼, 상품 카드, 장바구니 행의 공간을 더 압축합니다.
- 모바일에서는 이미지가 찌그러지지 않도록 `width: 100%`, `height: auto`, `object-fit` 설정을 유지해야 합니다.
- 상품 상세 이미지처럼 긴 세로 이미지는 원본 비율 유지가 가장 중요합니다.

## 8. 현재 구현 상태와 제한
- 현재는 정적 HTML/CSS/JS 기반입니다.
- 실제 상품 필터링, 정렬, 페이지네이션 기능은 미구현입니다.
- 장바구니 수량 변경은 화면 표시만 변경하며 합계 금액 재계산은 없습니다.
- 장바구니 삭제는 DOM 행 제거와 개수 표시 변경만 처리합니다.
- 위시리스트 선택/삭제/장바구니 담기 기능은 실제 데이터와 연결되어 있지 않습니다.
- 로그인/회원가입은 UI만 구현되어 있고 인증/가입 API는 없습니다.
- 검색 버튼은 UI만 있으며 검색 기능은 없습니다.
- 결제, 주문, 네이버 페이 연동은 없습니다.

- 상품 필터와 정렬 실제 동작 연결
- 상품 목록 페이지네이션 실제 데이터 연결
- 상품 상세 옵션 선택 상태와 재고 데이터 연결
- 장바구니 실제 데이터 저장 및 금액 재계산
- 관심상품 실제 데이터 저장 및 삭제 기능
- 로그인/회원가입 실제 기능 연동
- Cafe24 상품 데이터 구조 연결
- 결제 및 주문 플로우 연결
- 접근성 개선: 키보드 포커스, 폼 에러 메시지, aria 상태값 보강
- 이미지 최적화: 용량 압축, lazy loading 검수, 필요 시 WebP 변환
- 공통 컴포넌트화: header, footer, product card, membership banner, form controls

## 10. 작업자 참고사항
- 브랜드명은 `PAVONIQ`로 통일되어 있습니다.
- 현재 고객센터/회사 정보는 샘플 값입니다.
- 디자인 시안과 퍼블리싱 구조를 맞추는 단계이며 운영 기능은 별도 개발이 필요합니다.
- 추후 운영 시 공통 컴포넌트화를 먼저 진행하면 유지보수 비용을 줄일 수 있습니다.
- Cafe24 스킨 적용 시 정적 링크와 샘플 데이터를 템플릿 변수로 치환하는 작업이 필요합니다.
- 기존 화면 구조를 수정할 때는 PC, 태블릿, 모바일 주요 breakpoint를 함께 확인해야 합니다.
